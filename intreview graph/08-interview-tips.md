# Graph Interview — Cross-Pattern Tips & Edge Cases

## Before You Code: 3 Questions

```
1. Directed or undirected?
   → Affects: cycle detection method, topo sort applicability, edge direction in adj list

2. Weighted or unweighted?
   → Unweighted = BFS for shortest path
   → Weighted (no neg) = Dijkstra
   → Weighted (with neg) = Bellman-Ford

3. Is the graph given explicitly or implicitly?
   → Explicit: adj list/matrix provided
   → Implicit: neighbors generated by transformation (Word Ladder, Open Lock)
```

---

## Universal Checklist

- [ ] Build adjacency list first, not matrix (unless dense/small)
- [ ] Always use a `visited[]` array — mark BEFORE pushing to queue/stack
- [ ] Handle disconnected graph: loop over ALL nodes as potential starts
- [ ] Directed graph cycle detection → 3-color DFS, NOT parent-tracking
- [ ] Undirected graph cycle detection → parent-tracking DFS OR DSU
- [ ] For grid problems: bounds check `r>=0 && r<m && c>=0 && c<n` before accessing
- [ ] Draw a small example (5 nodes, 5 edges) and trace by hand

---

## Complexity Reference

| Algorithm | Time | Space | Notes |
|-----------|------|-------|-------|
| DFS / BFS | O(V+E) | O(V) | V = nodes, E = edges |
| Topological Sort (Kahn's) | O(V+E) | O(V) | Only on DAGs |
| DSU (path+rank) | O(α(n)) | O(V) | Practically O(1) per op |
| Dijkstra | O((V+E) log V) | O(V) | Non-negative weights only |
| Bellman-Ford | O(V·E) | O(V) | Handles negative edges |
| Floyd-Warshall | O(V³) | O(V²) | All-pairs, small graphs |
| Kruskal's MST | O(E log E) | O(V) | Sort edges + DSU |
| Prim's MST | O((V+E) log V) | O(V) | Min-heap |
| Bipartite Check | O(V+E) | O(V) | BFS 2-coloring |
| Grid DFS/BFS | O(m·n) | O(m·n) | m×n grid |

## Pattern Recognition Cheat Sheet

```
"Is there a path from A to B?"               → DFS/BFS
"Minimum HOPS from A to B?"                  → BFS (unweighted)
"Minimum COST from A to B?"                  → Dijkstra (no neg) / Bellman-Ford
"All pairs shortest paths?"                  → Floyd-Warshall
"Count connected components?"                → DFS loop + count / DSU
"Cycle in undirected graph?"                 → DSU (unite fails) / DFS parent track
"Cycle in directed graph?"                   → DFS 3-color (white/gray/black)
"Order tasks with dependencies?"             → Kahn's Topological Sort
"Is valid ordering possible?"                → Topo sort + cycle check
"Merge groups, is A same group as B?"        → DSU
"Connect all nodes cheapest?"                → MST (Kruskal's)
"Split into 2 groups, no intra edges?"       → Bipartite BFS 2-color
"Grid: count islands/regions?"               → Grid DFS
"Grid: shortest path?"                       → Grid BFS
"Grid: shortest path with cost?"             → Dijkstra on grid
"Multiple sources → expand outward?"         → Multi-source BFS
"Derive order from constraints?"             → Topological sort (Alien Dictionary)
"Use every edge exactly once?"               → Hierholzer's Eulerian path
```

## Top 5 Most Common Graph Mistakes in Interviews

### 1. Not marking visited before pushing to queue
```cpp
// Wrong (visited after pop → same node pushed multiple times)
q.push(start);
while (!q.empty()) {
    int u = q.front(); q.pop();
    visited[u] = true;  // too late

// Correct
visited[start] = true;
q.push(start);
```

### 2. Using parent-tracking for directed graph cycle detection
```
Undirected: track parent to ignore the edge you came from
Directed: need 3 colors (unvisited/in-stack/done) — parent doesn't work for DAGs
```

### 3. Dijkstra with max-heap instead of min-heap
```cpp
// Wrong: default priority_queue is MAX-heap
priority_queue<pair<int,int>> pq;

// Correct: use greater<> for MIN-heap
priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;
```

### 4. Topological sort direction for prerequisites
```cpp
// Course schedule: "to take course a, must first take course b"
// means: edge from b → a (b comes before a)
adj[b].push_back(a);  // NOT adj[a].push_back(b)
```

### 5. Grid bounds check order
```cpp
// Wrong: check grid[nr][nc] before checking bounds → out of bounds access
if (grid[nr][nc] != '1') ...

// Correct: bounds first
if (nr < 0 || nr >= m || nc < 0 || nc >= n) continue;
if (grid[nr][nc] != '1') continue;
```

## Things to Mention Without Being Asked

1. **Adjacency list vs matrix:** "I'll use an adjacency list since the graph may be sparse — matrix would be O(V²) space."
2. **Disconnected graph:** "I'm looping over all nodes since the graph might be disconnected."
3. **Visited before push:** "I mark visited before pushing to the queue to prevent duplicate visits."
4. **Directed vs undirected cycles:** "Since this is directed, I need 3-color DFS — the parent trick only works for undirected."
5. **BFS vs Dijkstra:** "Since edges are unweighted, BFS gives the correct shortest path in O(V+E). No need for Dijkstra here."
